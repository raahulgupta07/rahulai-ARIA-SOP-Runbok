## What it is
Bilingual (EN + Burmese) single-agent brain over company SOPs/policies (docs +
images). Vectorless RAG (PageIndex + Postgres FTS/trigram, NO embeddings),
vision page-reading at ingest, answer-with-page-image, memory + upload
self-learning. Multi-method auth, per-user chat history, streaming answers.
Packaged OpenWebUI-style: one FastAPI process serves API (`/api/*`) + SvelteKit
SPA on one port. Runs as one Docker image + postgres.

## Async ingest (v1.4.0 — replaces old sync /upload)
- **Flow:** `/upload` saves file → storage **inbox** + INSERTs `docs` row `status='queued'` → returns **instantly** `{doc_id,status}`. Daemon **worker** (`worker.py`, started in main lifespan) claims queued docs via `UPDATE … WHERE id=(SELECT id … status='queued' … FOR UPDATE SKIP LOCKED LIMIT 1) RETURNING` (atomic, autocommit-safe), runs `ingest.process_doc(doc_id, src, name)`, flips `ready`/`failed`, `storage.move_out(key, processed|failed)`.
- **process_doc** (was `ingest_file` body): writes live progress — render 10→30, vision per-page 30→80 (`preread_pages(pages, on_page=cb)` via `as_completed`), tree 85, done 100. **Rerun-safe**: DELETEs old pages/nodes for that doc first, UPDATEs the existing row (no new INSERT). Page-image dir keyed `doc_{id}`. `ingest_file` kept as thin sync wrapper (scripts/back-compat).
- **storage.py** abstraction: `save_to_inbox/inbox_path/move_out/list_inbox/original_filename`, `STORAGE=local|s3` (s3=NotImplementedError stub = Phase 8). Dirs `data/{inbox,processed,failed}` in config + ensure_dirs.
- **watcher.py** (started in lifespan): polls `storage.list_inbox()` every `WATCHER_POLL_SECONDS`, INSERTs queued row for any key with no `docs.storage_key` match → files dropped straight into inbox (bulk_upload.py now just copies there) auto-ingest. Dedup = storage_key (HTTP uploads already rowed → skipped). **Single watcher thread** (multiple would race inserts).
- **Crash recovery:** `worker.boot_sweep()` requeues `processing`→`queued` on boot.
- **Gating:** `retrieve.search_pages` both queries filter `d.status='ready'` (queued/processing/failed never answer). `/documents` returns `status,progress,pages_done,error,ready_at`; `/documents/{id}/retry` re-queues failed.
- **vision.py:** client `timeout=VISION_TIMEOUT(90s)` + `max_retries=2` so one stalled OpenRouter call can't hang ingest. `VISION_WORKERS=6` concurrent/doc.
- **Knobs:** `INGEST_WORKER`(on), `INGEST_WORKERS`(parallel docs), `INGEST_POLL_SECONDS`, `INGEST_WATCHER`, `WATCHER_POLL_SECONDS`, `VISION_TIMEOUT/WORKERS`, `STORAGE`.
- **Frontend (brain/+page.svelte):** upload → instant queued row; `$effect` polls `/documents` every 2s while any queued/processing, auto-stops idle; status badges + progress bar; non-ready rows gated (no inspector open); failed → ↻ retry (`api.retryDoc`). **Known cosmetic:** progress sits at 85% during PageIndex tree build (~50s, no sub-progress).

## Chat / answer pipeline
- **Retrieve** (`search_pages`, vectorless): one indexed SQL, score = `ts_rank(fts) + 2×node_section + 0.5×trigram + 0.3×ilike`; LLM query expansion (EN+MY synonyms); fallback recent-doc pages (also `status='ready'` gated). `RETRIEVE_K` default **8**. Returns page body (`text_full`) **+ compiled markdown** when present (LEFT JOIN `doc_pages_md`, served instead of raw when `CHAT_USE_WIKI=1`) + image_path.
  - **Query-side answer-quality pass (2026-06-12-latest — NO doc reprocessing; goal = better+detailed answers without an ingest enrichment pass, which dies on 500-pg docs):**
    - **`_expand` sharpened** — was "give 8 generic synonyms"; now a system-prompted call that **expands acronyms↔short forms** (`adj code`↔`adjustment reason code`, `GLD`→`Gold Central`), predicts likely **on-screen / menu / table / field / button names** a screenshot would show, **action synonyms** (add/create/insert; disable/deactivate), + Burmese equivalent. Up to 12 terms. ilike phrase filter widened 40→**50 chars / 10 phrases**. Fixes the real recall miss (vectorless FTS = literal match; casual wording ≠ formal doc wording). Verified "how to add adj code" → matched formal "Adjustment Reason Code Adding Process" p.4/5/6/7. Env `RETRIEVE_EXPAND`.
    - **Relevance RERANK `RETRIEVE_RERANK=1` (`_rerank`)** — FTS recall is broad but ordered by keyword frequency, not relevance. Over-fetch `RERANK_POOL=16` candidates → one small `temperature=0` LLM call ranks by how well each page ANSWERS the question (over 220-char snippets) → keep best `k`. Runs **BEFORE depth-fill** so depth-fill extends the page the user actually needs. **Fail-soft** → backfills/keeps FTS order on any error. Verified "disable a user account" → "User Creation & Disable p.7" ranked first.
  - **`text_full` cap = 16000** (was 4000 — the 4k cap was the ROOT CAUSE of thin answers; dense pages up to 21k chars were chopped 80% before the model saw them). Don't lower it.
  - **Depth-fill `RETRIEVE_DEPTH=5` (2026-06-12)** — 2nd ROOT CAUSE of generic answers: the per-page ranker spread all K hits across many docs' keyword-dense **TITLE pages** (1 each), so the body pages where the actual steps live never surfaced → model honestly said "no steps here, please upload." Fix: after ranking, pull the TOP doc's best-matching **body** pages (relevance-ordered, same tsq, skips already-have) and append. Verified: "automatic replenishment" went from "please upload" → full numbered CAR procedure citing p.4/p.5/p.11. Env `RETRIEVE_DEPTH` (compose passthrough); raise for very long SOPs.
- **Answer** (`agent.py`, 2-speed):
  - **quick** (default): feed FULL labeled page-text blocks (+ **conversation history**, last 3 Q/A) → ONE **streamed** OpenRouter call (`max_tokens=8000`), no images, no tool loops → fast + detailed.
  - **deep**: Agno agent (`max_tokens=8000`) + page **images** + toolbelt. **Deep does NOT re-run vision per query** — reads stored/compiled text; images only as citation context.
  - RULES force: answer in QUESTION language, extract every step/code/path exactly, structured, **inline `[p.N]` citations**, end `PAGES: <ids>`. **(2026-06-12-latest)** + quote exact on-screen values/screen/table/button names verbatim, **list ALL methods/screens/roles** when the SOP gives more than one (each labelled), render source tables as markdown, use markdown structure (bold codes/screens).
- **Conversation memory (2026-06-12)** — quick mode was STATELESS (each `/ask` ignored prior turns → "more details" answered as a new standalone Q). Fix: `stream_reply(...,history=)` injects last 6 messages (bot→assistant) before the current turn. **Retrieval is follow-up-aware**: `routes._is_followup(q)` (≤6 words OR cue regex more/detail/why/that/it/above…) → `_search_query()` folds the prior user question into the search so a follow-up stays on-topic.
- **Stream endpoint** `POST /api/ask/stream` → NDJSON lines `meta`→`token`*→`done{pages,title,clean}`. Persists user+bot messages, auto-titles first turn. Source `pages[]` now carry **`doc_id`** (for the citation drawer's "Open full document").
- **BLIND-SPOT SELF-CORRECTION LOOP (2026-06-13, Agentic-RAG)** — compose pass-1 → parse PAGES **STRICTLY** `parse_pages(acc, [])` (PAGES:none → `[]`; KILLS the old seed-fallback that cited ALL seed ids for a sourceless answer, masking blind spots). If ungrounded → ONE wider re-retrieve (`search_pages k=min(20,(k or DEFAULT_K)*2+4)`, merge only FRESH pages) → emit `_step(...,reset=True)` (client `onStep obj.reset`→`m.text=''` wipes attempt-1) → re-compose. Still ungrounded → REPLACE with "⚠️ couldn't find this in the runbooks… verify" banner + `cited=[]` + `notify.emit("audit","Blind spot…","warn")`. **Makes the dashboard/audit blind_spots metric REAL** (empty citations now detectable). Flag `BLINDSPOT_LOOP` env default `1` (off=legacy seed-fallback). `_step()` has `reset` kwarg; inline `_compose(pages)` gen yields `('tok',d)*`/`('acc',full)`. NEXT (#2): per-fact verify pass.
- **Conversations** per user (left rail): grouped by recency, LLM auto-title, rename/delete, ownership enforced.

## Compiled "wiki" layer (2026-06-12, Karpathy second-brain — vision ONCE, text forever)
- **Why**: vision/OCR runs only at ingest; chat never re-scans the file. The fix for thin answers = compile each page's raw vision text into **clean markdown ONCE**, store it, and let chat read the pre-digested markdown — so even flash-lite answers in full (tables render as md tables). **Did NOT change the chat model.**
- **`app/compile_wiki.py`** — `compile_doc(doc_id)`: per page, one `COMPILE_MODEL` LLM pass = "REFORMAT, never summarise" (tables→md tables, every step/code/SQL/value/Burmese kept) → `doc_pages_md(doc_id,page_no,md,chars)`; concat + short summary → `doc_wiki(doc_id,title,summary,md,key_facts)`. Fail-soft: any error → raw text kept (never lose content).
- **Ingest hook** (`ingest.process_doc`, gated `COMPILE_ON_INGEST=1`) compiles after vision. **Chat** (`retrieve.search_pages`, gated `CHAT_USE_WIKI=1`) serves `doc_pages_md.md` per page, **per-page raw fallback** if uncompiled.
- **Backfill** `scripts/compile_all.py` (run with `PYTHONPATH=.`) — recompiles all ready docs from STORED vision text (no re-vision/re-upload), idempotent. **Done: 31/31 docs, 271/271 pages.**
- Flags in `.env` + compose env block: `COMPILE_ON_INGEST=1`, `CHAT_USE_WIKI=1`, `COMPILE_MODEL` (empty→CHAT_MODEL). Reversible (flag off → raw text).
- **NOT done (Phase 3/4)**: cross-doc entity/index pages (would fix "CAR defined 3 ways" drift across SOPs) + lint/contradiction → Audit.

## Self-learning
1. Upload docs (instant). 2. `POST /memory` facts → **always injected** into every answer.
3. **Agent auto-writes facts mid-chat (v1.5.0, Scout-style write-back, NO ML)** — `app/learn.py`:
   - **Reactive + gated.** After each `/ask/stream` answer, `maybe_learn(q, answer, user_email)` runs. Gate first (`wants_to_teach` regex, EN + Burmese `မှတ်ထား/မှတ်သား/မမေ့`) → **zero extra LLM cost on normal questions**; only teaching turns hit the extractor.
   - Trigger words: remember / note that / from now on / always / correction / i meant / the correct value is / save this / add to brain / fyi …
   - Extractor = small CHAT_MODEL call → JSON `{save, fact}`; conservative (chit-chat/questions → `save=false`). On save → `add_memory(fact, source='chat', created_by=email, status='pending')` → emits **"💡 Learned something new — saved for review"** step + bell notify.
   - `memory.source` = `human` (taught via UI / `POST /memory`) vs `chat` (agent learned). `memory.created_by` = user email. Brain Facts tab: 🤖 + "learned in chat" chip vs 💡; inspector "Learned fact … from {email}".
   - DEEP agent still has the `remember_fact` Agno tool (also `source='chat'`, also `status='pending'`).
4. **Review gate (v1.6.0, AIS-OS Intern Rule)** — chat-learned facts land **`status='pending'`** and DON'T affect answers (`memory_facts()` injects ACTIVE-only) until an admin **Approves** in Brain → Facts (`POST /memory/{id}/approve|reject`). Hand-taught (UI `POST /memory`) facts are `active` immediately. Once active, **a fact OVERRIDES the docs on conflict** (agent RULES PRECEDENCE block; answer follows fact + notes SOP outdated).
5. **Fact freshness (v1.6.0)** — `touch_used_facts(answer)` post-turn bumps `cited_count`/`last_cited_at` for active facts whose wording (key, or first ~6 words of value, ≥4 chars) appears in the answer. Coverage Audit flags active+never-used+14d-old as **stale** → Retire.
