## Async ingest (v1.4.0 — replaces old sync /upload)
- **Flow:** `/upload` saves file → storage **inbox** + INSERTs `docs` row `status='queued'` → returns **instantly** `{doc_id,status}`. Daemon **worker** (`worker.py`, started in main lifespan) claims queued docs via `UPDATE … WHERE id=(SELECT id … status='queued' … FOR UPDATE SKIP LOCKED LIMIT 1) RETURNING` (atomic, autocommit-safe), runs `ingest.process_doc(doc_id, src, name)`, flips `ready`/`failed`, `storage.move_out(key, processed|failed)`.
- **process_doc** (was `ingest_file` body): writes live progress — render 10→30, vision per-page 30→80 (`preread_pages(pages, on_page=cb)` via `as_completed`), tree 85, done 100. **Rerun-safe**: DELETEs old pages/nodes for that doc first, UPDATEs the existing row (no new INSERT). Page-image dir keyed `doc_{id}`. `ingest_file` kept as thin sync wrapper (scripts/back-compat).
- **storage.py** abstraction: `save_to_inbox/inbox_path/move_out/list_inbox/original_filename`, `STORAGE=local|s3` (s3=NotImplementedError stub = Phase 8). Dirs `data/{inbox,processed,failed}` in config + ensure_dirs.
- **watcher.py** (started in lifespan): polls `storage.list_inbox()` every `WATCHER_POLL_SECONDS`, INSERTs queued row for any key with no `docs.storage_key` match → files dropped straight into inbox (bulk_upload.py now just copies there) auto-ingest. Dedup = storage_key (HTTP uploads already rowed → skipped). **Single watcher thread** (multiple would race inserts).
- **Crash recovery:** `worker.boot_sweep()` requeues `processing`→`queued` on boot.
- **Gating:** `retrieve.search_pages` both queries filter `d.status='ready'` (queued/processing/failed never answer). `/documents` returns `status,progress,pages_done,error,ready_at`; `/documents/{id}/retry` re-queues failed.
- **vision.py:** client `timeout=VISION_TIMEOUT(90s)` + `max_retries=2` so one stalled OpenRouter call can't hang ingest. `VISION_WORKERS=6` concurrent/doc.
- **Knobs:** `INGEST_WORKER`(on), `INGEST_WORKERS`(parallel docs), `INGEST_POLL_SECONDS`, `INGEST_WATCHER`, `WATCHER_POLL_SECONDS`, `VISION_TIMEOUT/WORKERS`, `STORAGE`.
- **Frontend (brain/+page.svelte):** upload → instant queued row; `$effect` polls `/documents` every 2s while any queued/processing, auto-stops idle; status badges + progress bar; non-ready rows gated (no inspector open); failed → ↻ retry (`api.retryDoc`). **Known cosmetic:** progress sits at 85% during PageIndex tree build (~50s, no sub-progress).

## Chat / answer pipeline

Returns page body (`text_full`) **+ compiled markdown** when present (LEFT JOIN `doc_pages_md`, served instead of raw when `CHAT_USE_WIKI=1`) + image_path.

Up to 12 terms. ilike phrase filter widened 40→**50 chars / 10 phrases**. Fixes the real recall miss (vectorless FTS = literal match; casual wording ≠ formal doc wording). Verified "how to add adj code" → matched formal "Adjustment Reason Code Adding Process" p.4/5/6/7. Env `RETRIEVE_EXPAND`.
    - **Relevance RERANK `RETRIEVE_RERANK=1` (`_rerank`)** — FTS recall is broad but ordered by keyword frequency, not relevance. Over-fetch `RERANK_POOL=16` candidates → one small `temperature=0` LLM call ranks by how well each page ANSWERS the question (over 220-char snippets) → keep best `k`. Runs **BEFORE depth-fill** so depth-fill extends the page the user actually needs. **Fail-soft** → backfills/keeps FTS order on any error.
