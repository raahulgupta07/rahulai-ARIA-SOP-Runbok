## What it is
Bilingual (EN + Burmese) single-agent brain over company SOPs/policies (docs +
images). Vectorless RAG (PageIndex + Postgres FTS/trigram, NO embeddings),
vision page-reading at ingest, answer-with-page-image, memory + upload
self-learning. Multi-method auth, per-user chat history, streaming answers.
Packaged OpenWebUI-style: one FastAPI process serves API (`/api/*`) + SvelteKit
SPA on one port. Runs as one Docker image + postgres.

## Async ingest (v1.4.0 ‚Äî replaces old sync /upload)
- **Flow:** `/upload` saves file ‚Üí storage **inbox** + INSERTs `docs` row `status='queued'` ‚Üí returns **instantly** `{doc_id,status}`. Daemon **worker** (`worker.py`, started in main lifespan) claims queued docs via `UPDATE ‚Ä¶ WHERE id=(SELECT id ‚Ä¶ status='queued' ‚Ä¶ FOR UPDATE SKIP LOCKED LIMIT 1) RETURNING` (atomic, autocommit-safe), runs `ingest.process_doc(doc_id, src, name)`, flips `ready`/`failed`, `storage.move_out(key, processed|failed)`.
- **process_doc** (was `ingest_file` body): writes live progress ‚Äî render 10‚Üí30, vision per-page 30‚Üí80 (`preread_pages(pages, on_page=cb)` via `as_completed`), tree 85, done 100. **Rerun-safe**: DELETEs old pages/nodes for that doc first, UPDATEs the existing row (no new INSERT). Page-image dir keyed `doc_{id}`. `ingest_file` kept as thin sync wrapper (scripts/back-compat).
- **storage.py** abstraction: `save_to_inbox/inbox_path/move_out/list_inbox/original_filename`, `STORAGE=local|s3` (s3=NotImplementedError stub = Phase 8). Dirs `data/{inbox,processed,failed}` in config + ensure_dirs.
- **watcher.py** (started in lifespan): polls `storage.list_inbox()` every `WATCHER_POLL_SECONDS`, INSERTs queued row for any key with no `docs.storage_key` match ‚Üí files dropped straight into inbox (bulk_upload.py now just copies there) auto-ingest. Dedup = storage_key (HTTP uploads already rowed ‚Üí skipped). **Single watcher thread** (multiple would race inserts).
- **Crash recovery:** `worker.boot_sweep()` requeues `processing`‚Üí`queued` on boot.
- **Gating:** `retrieve.search_pages` both queries filter `d.status='ready'` (queued/processing/failed never answer). `/documents` returns `status,progress,pages_done,error,ready_at`; `/documents/{id}/retry` re-queues failed.
- **vision.py:** client `timeout=VISION_TIMEOUT(90s)` + `max_retries=2` so one stalled OpenRouter call can't hang ingest. `VISION_WORKERS=6` concurrent/doc.
- **Knobs:** `INGEST_WORKER`(on), `INGEST_WORKERS`(parallel docs), `INGEST_POLL_SECONDS`, `INGEST_WATCHER`, `WATCHER_POLL_SECONDS`, `VISION_TIMEOUT/WORKERS`, `STORAGE`.
